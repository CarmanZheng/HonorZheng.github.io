### 第二部分 浮动和布局

#### 1. 浮动 

浮动用到了css里面的float属性

| 属性值  | 描述                     |
| :------ | ------------------------ |
| none    | 元素不浮动，标准流布局   |
| left    | 元素浮动在父级元素的左边 |
| right   | 元素浮动在父级元素的右边 |
| inherit | 继承父级元素的float属性  |

##### 1.1 浮动可以是块级元素在同一行显示

```html
<div style="float:left">
    <div style="background:red">
        块级元素A
    </div>
    <div style="background:yellow">
        块级元素B
    </div>            
</div>
```

##### 1.2 浮动使行内元素具有块级元素特征

tip：标准流排版中，行内元素不能设置**宽、高、上下边距**；

当设置行内元素为浮动时，其具有块级元素的特征，可以设置**宽、高和四个方向的边距**

```html
<style>
    span{
        float:left;
        width:80px;
        height:80px;
        margin:15px;
        padding:15px;
    }
</style>

<div>
    <span>span1</span>
    <span>span2</span>
</div>
```

##### 1.3浮动元素不设置宽高时，宽高由内容撑开

##### 1.4父级元素塌陷问题

​	使用伪元素解决

```html
<style>
    .father:after{
        content:'',      /*没有内容*/
        display:block,  /*块级元素*/
        clear:both     /*清除浮动*/
    }
</style>

```

触发BFC【Block Formatting Context 块级格式上下文】解决塌陷

```html
<style>
    .father{
        width:330px,
        display:inline-block, /*设置行内块元素触发BFC解决塌陷*/         
    }
</style>
```

```html
<style>
    .father{
        float:left, /*父级元素设定浮动触发BFC解决塌陷*/
    }
</style>
```

```html
<style>
    .father{
        overflow:hidden, /*父级元素设定overflow触发BFC解决塌陷*/
    }
</style>
```



### 第三部分   定位

| 属性值   | 描述                                                         |
| -------- | ------------------------------------------------------------ |
| static   | 静态定位，默认值，元素按照标准流进行布局                     |
| relative | 相对定位，相对于自身位置进行位置偏移                         |
| absolute | 绝对定位，对象从文档流中脱离出来，相对于最近的一个已定位的【相对/绝对】祖先级元素定位 |
| fixed    | 固定定位，根据浏览器窗口来进行位置偏移                       |

#### 1.1 静态定位

元素按照标准流从上至下一次排列

#### 1.2 相对定位

 ```css
postion:relative	
 ```

1. 只设置相对定位，不设置偏移量【left，right，top，buttom】，元素位置不发生变化
2. 设置偏移量，原位置会保留下来，即**不脱离文档流**
3. 偏移后的元素，元素层级提升，至于同类型元素之上【可以理解为z-index增大】
4. 新位置为相对于原位置的移动【left，right，top，buttom】

```html
<style>
    .son1,.son2.son3{
       width:100px;
       height:100px;
       margin:5px;
       display:inline-block;
       border:1px dashed black;
    }
    
    .son1{
        background:#0FF;
    }
    .son2{
        background:#9CF;
        position:relative;
        left:50px;
        top:50px;
    }
    .son3{
        background:#3FF;
    }
</style>
<body>
    <div class="son1">框1</div>
    <div class="son2">框2</div>
    <div class="son3">框3</div>
</body>
```

#### 1.3 绝对定位

绝对定位是相对于距离自己最近的有定位的祖先元素进行定位或者相对于最外层的包含框的定位。

```css
postion:absolute	
```

1. 只给元素设置绝对定位，不设置偏移值，元素还在原来位置上，但是**会脱离文档流**【位置会被后面的元素占领】
2. 绝对定位也会提升偏移元素的层级
3. 根据最近的有定位的祖先元素来计算坐标，如果所有的祖先元素都没有定位，那么根据最外层的包含框来计算自己的坐标。
4. 不设置宽度时，宽度由内容撑开。
5. 绝对定位**使得行内元素变成块级元素**，可以设置宽高和上下边距
6. 父元素设置绝对定位，触发BCF，有效的解决塌陷的问题。

```html
<style>
    .father{
        position:absolute;
        width:330px;
        border: 1px dashed #0ff;
    }
    .son1{
        float:left; /*左浮动*/
        width:100px;
        background:green;
    }
    .son2{
        float:right;/*右浮动*/
        width:100px;
        background:red;
    }
</style>

<body>
    <div class="father">
        <div class="son1">左浮动 </div>
        <div class="son2">右浮动 </div>
    </div>
</body>
```

#### 1.4 固定定位

固定定位，是指相对于浏览器可视窗口进行的定位。位置固定，不会随网页的移动而移动。

```css
position:fixed
```

1. 提升层级，脱离文档流
2. 固定在可视区的某个位置，不随滚动条变化
3. 根据浏览器视口进行位置设定【left，top，right，buttom】
4. 使行内元素具有块级元素，可以设置宽高和上下边距
5. 触发BFC
